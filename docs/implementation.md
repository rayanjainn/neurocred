# App-Wide Implementation Matrix: FinTwin Architecture

This matrix correlates the theoretical architecture layers of FinTwin to the exact backend systems, frontend pages, and UI components found in the repository.

| Module | Feature Set | Backend Implementation | Frontend UI / Pages | Status & TODOs |
| :--- | :--- | :--- | :--- | :--- |
| **1. Multi-Source Signal Ingestion** | Bank transactions, UPI logs, SMS alerts, EMI schedules, open-banking feeds | `src/ingestion/` (Simulated data pipelines extracting open-banking feeds) | `frontend/app/msme/report/page.tsx` (Underlying data feed) & `TimeSeriesPanel.tsx` | **Implemented**: Mocked aggregation for UPI, EWB.<br>**TODO**: Live production SMS parsing and realtime external bank sync. |
| **2. Event Stream Processor** | Real-time typed financial event classification and sliding-window aggregation | `src/classifier/` & `src/features/` (Window aggregations) | `frontend/components/TimeSeriesPanel.tsx` (Shows aggregated window timelines inside Twin/Dashboard pages) | **Implemented**: Stream logic is mapped.<br>**TODO**: Live Apache Kafka/Redis streaming implementation. |
| **3. Behavioural Feature Engine** | Spending volatility, income stability, peer cohort benchmarking, trend detection | `src/features/` (Mathematics for volatility and stability signals) | `frontend/app/msme/report/page.tsx` (Score metrics breakdown showing volatility & stability scores) | **Implemented**: Volatility & trend detection math shown on credit reports.<br>**TODO**: Real-time peer cohort benchmarking view. |
| **4. Audit Repository & Dashboard** | Full-stack live dashboard, what-if simulation, regulatory audit export | `src/api/` & `src/strategy/` | `frontend/app/dashboard/` & PDF/JSON Export Buttons in `msme/twin/page.tsx` | **Implemented**: Native browser PDF builder and localized JSON state snapshots. |
